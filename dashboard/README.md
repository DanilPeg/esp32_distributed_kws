# Веб-дашборд агрегатора-мастера

Лёгкий веб-интерфейс для отображения событий распределённой системы распознавания голосовых команд в реальном времени. Построен по клиент-серверной схеме на основе FastAPI; источник данных — построчные журналы событий формата JSONL.

## Состав файлов

| Файл | Назначение |
|------|------------|
| `app.py` | Основной модуль FastAPI-приложения. Эндпоинты получения снимка состояния, потокового канала Server-Sent Events, отдачи статических ресурсов. |
| `loaders.py` | Чтение JSONL-журналов из каталогов `notes/Journal/hash_kws_telemetry/`, `hash_kws_fusion/`, `hash_kws_cluster/`. Поддержка инкрементальной подгрузки изменений в реальном времени. |
| `enrich.py` | Обогащение событий: применение временного окна голосования (объединение последовательных одинаковых распознаваний от одного узла), классификация типов событий, нормализация меток времени. |
| `paths.py` | Резолвер путей к каталогам журналов и шаблонам. Допускает переопределение через переменные окружения. |
| `templates/` | Шаблоны Jinja2 для главной страницы и вспомогательных представлений. |

## Архитектура

Дашборд работает как локальный веб-сервер на порту 8765 по умолчанию. Главная страница содержит четыре функциональных блока:

- верхняя hero-полоса с карточками последних распознанных команд и счётчиком повторений в формате `×N`;
- центральная таблица всех событий за последние пять минут с возможностью прокрутки вглубь истории;
- боковая панель состояния узлов с маркерами онлайн/офлайн, временем последнего heartbeat и числом срабатываний;
- нижняя секция с графиками кумулятивного числа срабатываний по каждому классу.

Поток обновлений в браузер организован через Server-Sent Events: при появлении новой записи в одном из JSONL-журналов сервер отправляет событие подключённым клиентам, веб-интерфейс обновляет соответствующий блок без полной перезагрузки страницы. Типичная задержка отображения после поступления события на узел — менее 100 мс.

## Запуск

Из корня репозитория:

```
python run_dashboard.py
```

Опциональные параметры:

- `--host 127.0.0.1` — адрес связывания (по умолчанию `127.0.0.1`).
- `--port 8765` — TCP-порт (по умолчанию `8765`).
- `--reload` — автоматическая перезагрузка при изменении кода (для разработки).
- `--log-level info` — уровень журналирования сервера uvicorn.

Веб-интерфейс доступен по адресу `http://127.0.0.1:8765/`.

## Временное окно голосования

Без специальной обработки одно произнесённое слово порождает несколько срабатываний модели подряд (поскольку расписание эпизодов запускает инференс несколько раз в течение длинного речевого эпизода), что приводит к визуальному дроблению hero-полосы. Для устранения этого эффекта в модуле `enrich.py` реализован механизм объединения: последовательные срабатывания с одной меткой класса от одного узла, попадающие в окно длиной 2 000 миллисекунд, представляются одной карточкой с счётчиком повторений. Длительность окна согласована со средней продолжительностью речевого эпизода и обеспечивает естественное восприятие интерфейса оператором.

## Зависимости

Установить пакеты:

```
pip install fastapi uvicorn jinja2 pyserial
```

## Принцип «только чтение»

Дашборд является потребителем артефактов: он читает JSONL-журналы и снимки состояния, формируемые мостом последовательного канала и симулятором кластера, но никогда не пишет в них. Это упрощает диагностику и позволяет одновременно запускать несколько экземпляров диспетчерского интерфейса, например на разных портах для удалённого наблюдения.
