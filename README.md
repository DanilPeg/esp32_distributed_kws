# Распределённая система распознавания голосовых команд на ESP32

Распределённая автоматизированная система обработки информации на базе сети микроконтроллеров ESP32-S3, выполняющая локальный нейросетевой инференс распознавания ключевых слов и обменивающаяся короткими сообщениями уровня решения по адаптированному протоколу ESP-NOW. Один из узлов либо отдельный персональный компьютер выступает в роли агрегатора-мастера и предоставляет диспетчерский веб-интерфейс с отображением событий распознавания в реальном времени.

## Содержание репозитория

| Каталог | Назначение |
|---------|------------|
| `firmware/` | Встроенное программное обеспечение узла. TensorFlow Lite for Microcontrollers, оптимизированные ядра esp-nn, аудио-фронтенд microfrontend, прикладной протокол ESP-NOW. Сборка под Arduino IDE 2.x с установленным ядром arduino-esp32 версии 3.0 и старше. |
| `training/` | Обучение моделей. Облачные ноутбуки на основе TensorFlow и Keras, рецепты обучения учительских и ученических моделей, дистилляция знаний, INT8-квантование. Целевой набор данных — Speech Commands версии 0.02. |
| `analysis/hash_ensemble/` | Эксперименты по распределённому ансамблевому инференсу: однородные и разнородные ансамбли моделей с хешированными весами, сопоставление алгоритмов агрегации решений, оценка стабильности голоса ансамбля. |
| `dashboard/` | Веб-дашборд агрегатора-мастера на основе FastAPI. Главная страница с временным окном голосования, таблицей событий и панелью состояния узлов. Источник данных — построчные журналы событий формата JSONL. |
| `scripts/` | Служебные скрипты: запуск распределённого демонстрационного сценария, мост последовательного канала к плате, генератор виртуальных пиров для агрегации, экспорт прошивки. |
| `run_dashboard.py` | Корневой запускатор веб-сервера дашборда. |

## Архитектура решения

Каждый узел сети располагает собственной моделью локального инференса и принимает решение о наличии события (произнесённой команды) самостоятельно, на основе локально собираемых данных. Единый централизованный планировщик отсутствует. Узел публикует короткое сообщение уровня решения в общий радиоканал по протоколу ESP-NOW; агрегатор объединяет апостериорные распределения нескольких узлов и формирует итоговое решение.

Архитектура построена как пятиуровневая модель:

- Уровень 0 — физическая среда и сенсоры (цифровой микрофон MEMS-типа, опциональная камера).
- Уровень 1 — захват и предобработка сигнала (DMA-захват аудиосигнала через интерфейс I2S, аудио-фронтенд microfrontend).
- Уровень 2 — локальный инференс (интерпретатор TensorFlow Lite for Microcontrollers с загруженной квантованной моделью, ядра esp-nn).
- Уровень 3 — координация (прикладной протокол поверх ESP-NOW, согласование решений с временным окном).
- Уровень 4 — визуализация и метрики (агрегатор-мастер на стороне ПК, веб-сервер FastAPI, веб-интерфейс).

## Аппаратная база

- Микроконтроллер: ESP32-S3 в исполнении модуля ESP32-S3-WROOM-1 с 8 МБ внешней PSRAM и 8 МБ flash-памяти.
- Микрофон: цифровой MEMS-микрофон INMP441 с интерфейсом I2S, частота дискретизации 16 кГц.
- Камера (опционально, для зрительного канала): модуль на базе сенсора OV5640.
- Управляющий компьютер: x86-машина с операционной системой Windows 11 либо Linux, среда Arduino IDE 2.x и интерпретатор Python 3.11 и старше.

## Сценарий запуска штатного режима

1. Прошить плату ESP32-S3 проектом `firmware/micro_speech_sim/micro_speech/` через Arduino IDE с включённой опцией внешней PSRAM (Tools → PSRAM → OPI PSRAM) и значением идентификатора узла `HASH_KWS_NODE_ID = 1`.
2. Подключить плату к компьютеру по интерфейсу USB.
3. На компьютере запустить распределённое демонстрационное приложение:

```
python scripts/run_distributed_demo.py --port COM5 --single-node
```

(подставить фактический COM-порт; флаг `--single-node` отключает виртуальные пиры — в дашборде отображается только реальная плата).

4. Открыть в браузере адрес `http://127.0.0.1:8765`.
5. При произнесении одного из поддерживаемых ключевых слов в микрофон в верхней части веб-интерфейса появится карточка распознанной команды.

Поддерживаемые ключевые слова: `yes`, `no`, `up`, `down`, `left`, `right`, `on`, `off`, `stop`, `go`.

## Требования и зависимости

- Python 3.11 и старше для скриптов и веб-сервера.
- Установленные пакеты `fastapi`, `uvicorn`, `jinja2`, `pyserial` для дашборда и моста последовательного канала.
- Установленные пакеты `tensorflow` версии 2.15 и старше, `tensorflow-model-optimization`, `numpy`, `librosa` для обучения моделей.
- Arduino IDE 2.x с ядром arduino-esp32 версии 3.0 и старше для сборки прошивки.
