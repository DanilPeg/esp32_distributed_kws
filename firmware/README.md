# Встроенное программное обеспечение узла

Прошивки для микроконтроллера ESP32-S3 в составе распределённой системы распознавания голосовых команд. Все проекты собираются в Arduino IDE 2.x с установленным ядром arduino-esp32 версии 3.0 и старше.

## Состав каталогов

| Каталог | Назначение |
|---------|------------|
| `micro_speech_sim/micro_speech/` | Основная прошивка узла. Захват аудиосигнала через интерфейс I2S, аудио-фронтенд microfrontend, локальный инференс модели распознавания ключевых слов на основе TensorFlow Lite for Microcontrollers, межузловой обмен по протоколу ESP-NOW, светодиодная индикация состояний, системная диагностика. |
| `hash_kws_runtime/` | Подключаемая библиотека рантайма инференса для моделей семейства HashedNet. Реализация селективного диспетчинга ядер esp-nn: свёртки 3×3 направляются на референсные скалярные ядра (обход известной ошибки SIMD-реализации), точечные свёртки 1×1 используют оптимизированный кернел `mult8_1x1` напрямую. |
| `i2s_mic_test/` | Утилита проверки работоспособности подключения цифрового микрофона INMP441 по интерфейсу I2S. Используется при первичной сборке стенда. |
| `i2s_mic_probe/` | Дополнительная диагностическая утилита для анализа сырого аудиопотока с микрофона: вывод гистограмм амплитуд, оценка уровня фонового шума. |

## Конфигурация прошивки

Параметры узла задаются на этапе компиляции в файле `experiment_config.h`:

- `HASH_KWS_NODE_ID` — целочисленный идентификатор узла в диапазоне 0–255.
- `HASH_KWS_USE_ESPNOW` — флаг включения межузлового обмена по ESP-NOW.
- `HASH_KWS_WIFI_CHANNEL` — номер общего канала Wi-Fi для всех узлов.
- `HASH_KWS_TELEMETRY_STREAM` — флаг включения вывода JSONL-событий в последовательный канал для подключения к диспетчерскому интерфейсу.
- `HASH_KWS_USE_OLED` — опциональная активация драйвера OLED-дисплея для локальной индикации состояния.

## Сборка и прошивка

1. Установить Arduino IDE 2.x.
2. Добавить поддержку плат ESP32 через менеджер плат: ссылка `https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json`.
3. Установить ядро arduino-esp32 версии 3.0 и старше.
4. Открыть файл `micro_speech_sim/micro_speech/micro_speech.ino`.
5. В файле `experiment_config.h` задать значения параметров узла.
6. Включить опцию внешней PSRAM: меню Tools → PSRAM → OPI PSRAM.
7. Выбрать целевую плату ESP32-S3 и подключённый COM-порт.
8. Скомпилировать и загрузить прошивку.
9. Открыть монитор последовательного канала со скоростью 115 200 бод и убедиться в успешном старте узла и публикации первого heartbeat-сообщения.

## Использование оптимизированных ядер esp-nn

Рантайм инференса в каталоге `hash_kws_runtime/` использует библиотеку esp-nn с применением селективного диспетчинга. На свёртках 3×3 (стартовом блоке и поканальных слоях) макросы диспетчинга переопределяются на референсные ядра суффикса `_ansi`, что обходит известную проблему SIMD-реализации. Для точечных свёрток 1×1 выполняется явный вызов кернела `esp_nn_conv_s8_esp32s3`. Подобная схема обеспечивает ускорение шага инференса в 2,3 раза относительно полностью скалярной реализации без потери корректности результата.
