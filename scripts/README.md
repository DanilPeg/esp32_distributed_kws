# Служебные скрипты

Хост-сторонние скрипты для запуска распределённого демонстрационного сценария, моста к плате ESP32, генератора виртуальных пиров для агрегации и экспорта прошивки.

## Состав

| Скрипт | Назначение |
|--------|------------|
| `run_distributed_demo.py` | Запускатор полного демонстрационного сценария одной командой. Поднимает мост последовательного канала к реальной плате, генератор виртуальных пиров для агрегации и веб-сервер дашборда. Поддерживает опциональные флаги `--single-node` (только реальная плата без виртуальных пиров) и `--no-dashboard` (для отдельно запущенного веб-сервера). |
| `hash_kws_serial_bridge.py` | Мост последовательного канала. Слушает плату ESP32 по указанному COM-порту, парсит JSONL-сообщения, нормализует поля и пишет события в каталог `notes/Journal/hash_kws_telemetry/node1/`. |
| `hash_kws_cluster_sim.py` | Генератор виртуальных пиров для демонстрации механизма согласования решений. На основе событий реальной платы (узел 1) формирует синтетические события виртуальных узлов 2 и 3 с управляемой вероятностью согласия и шумом по оценке уверенности. Записывает результирующие fusion-решения в каталог `notes/Journal/hash_kws_fusion/`. |
| `hash_kws_dual_audio_fusion.py` | Подключаемый модуль с реализацией алгоритма согласования решений двух (или более) узлов с временным окном голосования. Используется как симулятором кластера, так и встроенным ПО узла. |
| `export_hash_kws_firmware.py` | Утилита экспорта обученной модели в C-массив для встраивания в прошивку. На вход подаётся бинарный файл модели в формате FlatBuffer, на выход формируются файлы `hash_model_data.cpp` и `hash_model_data.h`. |

## Сценарии запуска

### Штатный режим: одна реальная плата + дашборд

```
python scripts/run_distributed_demo.py --port COM5 --single-node
```

Подставить фактический COM-порт. Скрипт запустит мост последовательного канала и веб-сервер дашборда. Виртуальные пиры отключены — в дашборде отображается только реальная плата.

### Режим с виртуальной агрегацией

```
python scripts/run_distributed_demo.py --port COM5
```

Дополнительно к реальной плате запускается генератор виртуальных пиров. В дашборде отображаются три узла; механизм согласования формирует объединённые решения по правилу большинства с временным окном.

### Только мост последовательного канала

```
python scripts/hash_kws_serial_bridge.py --port COM5 --node-id 1
```

Подходит для отладки прошивки: события поступают в каталог журналов, но веб-сервер и виртуальные пиры не запускаются.

### Только генератор виртуальных пиров

```
python scripts/hash_kws_cluster_sim.py --watch
```

Используется для проверки работы дашборда без реального оборудования. Скрипт генерирует синтетические распознавания каждые 0,5 секунды.
